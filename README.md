## Structure

```
├── README.md               => github readme file (often autogenerated from `README.txt`)
├── README.txt              => wordpress.org format readme.txt
├── app                     => main php code
│   └── README.md           => readme for /app dir
├── assests                 => static files e.g. css, js, fonts, images
│   └── README.md           => readme for /assests dir
│   ├── css                 => css, scss, sass, less files
│   │   └── README.md       => readme for /css dir
│   ├── fonts               => font files e.g. woff
│   │   └── README.md       => readme for /fonts dir
│   ├── img                 => image files
│   │   └── README.md       => readme for /image
│   └── js                  => js files e.g. js, coffeescript
│       └── README.md       => readme for /js
├── bin                     => any binaary dependency, output zip files
│   └── README.md           => readme sources, usage
├── build                   => build scrips e.g. grunt, shell scripts,
│   └── README.md           => readme for build scripts
├── languages               => translations
│   └── README.md           => readme for readme
├── lib                     => lib e.g. rt-lib
│   └── README.md           => readme lib
├── sample-plugin.php       => plugin entry point - contains header
├── templates               => templates (php files but ususally front end, override by themes)
│   └── README.md           => readme templates
├── tests                   => test cases
│   ├── README.md           => readme for overall test
│   ├── functional          => end-to-end, functional, selenium, nightwatch
│   │   └── README.md       => readme how to run
│   └── unit                => unit test cases phpunit
│       └── README.md       => readme how to run
└── vendor                  => 3rd party lib - ID3 tag, php lib, jquery plugins
    └── README.md           => readme for sources, credits
